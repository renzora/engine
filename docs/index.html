Game Engine Documentation

Introduction

Welcome to the documentation for our Game Engine. This guide will walk you through the key features and functionalities of the engine that powers Renzora, and provide examples on how to use it effectively.

Installation

To get started with the Renzora Game Engine, you'll need to set up a few tools and follow the steps outlined below. The installation is tailored for Windows WSL (Windows Subsystem for Linux) but can be adapted to other environments.

Requirements

Docker Desktop

WSL (Windows Subsystem for Linux)

GitHub Desktop

MongoDB Software

Installing on Windows WSL

Follow these steps to install the Renzora Game Engine on Windows WSL:

Watch the installation demo: Installation Demo

Open your terminal of choice (make sure Docker Desktop is running) and enter the WSL environment: wsl

Run the following command to install the Renzora Game Engine: curl -L renzora.net/dev | bash

Start Server

Ensure Docker Desktop is running and you are in the WSL environment. Then, start the Renzora server with the following command: renzora

Notes

If you encounter a "WebSocket connection to 'wss://localhost:3000/' failed" error, it's due to the local dev server using a self-signed SSL certificate. To bypass this, visit https://localhost:3000. You'll see a warning that the connection is not private. Click "Advanced" and then "Proceed anyway". You can close the window immediately afterward, then refresh the website at http://localhost, and the WebSocket should now connect properly.

Assets

The assets object is responsible for managing game resources such as images, audio files, and JSON data. It handles the preloading, storage, and retrieval of these assets to ensure they are available when needed during gameplay.

Asset Types

The Renzora Game Engine supports the following types of assets:

  • Images: Used for sprites, tiles, and other visual elements. Supported formats include .png, .jpg, .jpeg, and .gif.
  • JSON Data: Contains structured data such as game objects, configurations, and maps.
  • Audio: Used for sound effects and background music. Supported formats include .mp3, .wav, and .ogg.

Preloading Assets

The preload method is used to load assets before they are needed in the game. This method ensures that all assets are available and fully loaded before the game starts, improving performance and preventing delays during gameplay. 


// Example of preloading assets
assets.preload([
    { name: 'playerSprite', path: 'img/sprites/player.png' },
    { name: 'backgroundMusic', path: 'audio/music/bg_music.mp3' },
    { name: 'gameConfig', path: 'json/gameConfig.json' }
], function() {
    console.log("All assets loaded");
    game.init();
});

Loading and Using Assets

After assets are preloaded, they can be accessed using the load method. This method retrieves the loaded asset by its name. 


// Example of loading and using an asset
const playerImage = assets.load('playerSprite');
game.ctx.drawImage(playerImage, 100, 100);

Render

The render object is responsible for all the rendering tasks in the Renzora Game Engine. It handles the drawing of background tiles, sprites, effects, and UI elements onto the game's canvas. This object also manages the rendering order, ensuring that all elements appear correctly on the screen.

Rendering Process

The rendering process involves several steps, including updating the game logic, rendering background and foreground tiles, sprites, and handling various effects such as lighting, weather, and particles.

Key Methods

  • parseRange(rangeString): Parses a string representing a range of numbers (e.g., "1-5") into an array of numbers. This is used for handling tile animations.
  • expandTileData(tileData): Expands tile data to handle animated frames or ranges of frames for tiles.
  • updateGameLogic(deltaTime): Updates the game state, including sprites, camera, game time, and other dynamic elements. This method is called in each game loop iteration.
  • renderBackground(viewportXStart, viewportXEnd, viewportYStart, viewportYEnd): Renders the background tiles within the visible viewport.
  • renderAll(viewportXStart, viewportXEnd, viewportYStart, viewportYEnd): Renders all elements (background tiles, sprites, etc.) within the visible viewport. It collects these elements in a render queue and sorts them by their z-index before drawing them.
  • renderLightingEffects(): Renders lighting effects, such as day/night cycles and dynamic lights.
  • renderWeatherEffects(): Renders weather effects like snow, rain, and fireflies.
  • handleDebugUtilities(): Renders debugging utilities like grid lines, collision boundaries, and walkable tiles if enabled.

Rendering Background Tiles

The renderBackground method is responsible for drawing the background tiles within the visible area of the game. It calculates the positions of the tiles based on the current camera position and viewport size, and draws them on the canvas. 


// Example of rendering background tiles
const backgroundTileCount = render.renderBackground(viewportXStart, viewportXEnd, viewportYStart, viewportYEnd);

Handling Lighting and Weather Effects

The renderLightingEffects and renderWeatherEffects methods manage the rendering of dynamic lighting and weather effects in the game. These methods ensure that lighting conditions and weather patterns are rendered correctly based on the game state and time of day. 


// Example of rendering lighting effects
render.renderLightingEffects();

// Example of rendering weather effects
render.renderWeatherEffects();
